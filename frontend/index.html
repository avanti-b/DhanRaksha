About Dhan₹aksha

Guarding every rupee with intelligent precision

Dhan₹aksha — meaning "Protection of Wealth" in Sanskrit — is an AI-powered fraud detection system built to shield financial transactions from the ever-evolving threat of digital fraud. Using state-of-the-art machine learning, we identify fraudulent patterns that rule-based systems miss entirely.

The problem we solve

India's digital payments ecosystem processes over ₹14 trillion monthly. Rule-based fraud filters are static — they cannot adapt to new attack patterns. They produce high false-positive rates, blocking genuine customers and eroding trust.

Dhan₹aksha uses supervised machine learning to learn fraud patterns from historical data, dramatically reducing both missed fraud and false alarms.

V1–V28 · PCA Features

ML pipeline · end to end
1

Data collection

Download creditcard.csv from Kaggle — 284,807 transactions, 30 features. V1–V28 are PCA-anonymised to protect cardholders.

2

Data cleaning

Remove nulls and duplicates. Exploratory Data Analysis — plot class distribution, amount histograms, fraud by hour, feature correlations.

3

Feature engineering & scaling

StandardScaler on Time and Amount. V1–V28 already normalised via PCA. 80/20 stratified train-test split preserves fraud ratio.

4

SMOTE · class balancing

Fraud is 0.17% of data. SMOTE synthesises fraud samples → balanced 50/50 training set. Applied ONLY on training data, never test.

5

Model training

Logistic Regression (baseline) and Random Forest (100 estimators, class_weight=balanced) trained on SMOTE data. n_jobs=-1 for speed.

6

Evaluation

Precision, Recall, F1, AUC-ROC, confusion matrices on held-out test set. All plots saved to /plots/. Metrics saved to metrics.json.

7

Deployment

Best model (RF) saved as fraud_model.pkl. Flask API (app.py) serves predictions at /api/predict. Frontend connects via fetch().
