ML pipeline — step by step
1

Data collection

Download creditcard.csv from Kaggle (284,807 rows, 31 features). V1–V28 are PCA-transformed for privacy.

2

Data cleaning (train_model.py)

Check for null values and duplicates. Remove noisy/invalid records. Perform EDA — plot distributions, class balance, and correlations.

3

Feature engineering

Apply StandardScaler to Time and Amount. V1–V28 already scaled. Add Hour feature for EDA. 80/20 stratified train-test split.

4

SMOTE (class balancing)

Fraud is only 0.17% of data. SMOTE generates synthetic fraud samples → balanced 50/50 training set. Applied ONLY on training data, not test data.

5

Model training

Train Logistic Regression (baseline) and Random Forest (100 estimators) on SMOTE-balanced data. Both use class_weight='balanced'.

6

Evaluation

Evaluate on untouched test set. Report Precision, Recall, F1, AUC-ROC, and confusion matrices. Save plots to /plots/.

7

Deployment (Flask API)

Best model (Random Forest) saved as fraud_model.pkl. Flask API (app.py) serves predictions at http://localhost:5000/api/predict.
